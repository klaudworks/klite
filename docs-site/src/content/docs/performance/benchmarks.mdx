## Setup

<LinkButton href="https://github.com/klaudworks/klite/blob/main/infra/bench/README.md" variant="minimal" icon="github">README</LinkButton>
<LinkButton href="https://github.com/klaudworks/klite/tree/main/cmd/klite-bench" variant="minimal" icon="github">Benchmark code</LinkButton>
<LinkButton href="https://github.com/klaudworks/klite/tree/main/infra/bench" variant="minimal" icon="github">Terraform</LinkButton>

Two `m7g.xlarge` (ARM, 4 vCPU, 16 GB) instances on AWS in the same availability zone (eu-west-1). One runs klite, the other runs the benchmark client. The machines are independent so the numbers include real network round-trips, not just loopback. Actual memory usage is closer to 6 GB, we chose these instances because AWS throttles EBS network throughput based on instance size, and a smaller instance cannot sustain 100 MB/s of writes to disk.

The benchmark produces 100,000 events per second (each ~1 KB, roughly 97 MB/s) and consumes them back, sustained for one full hour. That's 360 million events total. Acks are set to 1, meaning klite explicitly acknowledges each event after writing it to disk. No fire-and-forget tricks to make the numbers look better.
