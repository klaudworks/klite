## Setup

Two `m7g.xlarge` (ARM, 4 vCPU, 16 GB) instances on AWS in the same availability zone. One runs klite, the other runs the benchmark client. Separate machines so the numbers include real network hops, not loopback. klite only uses ~6 GB of memory. We need this instance size not for RAM but because AWS throttles disk throughput on smaller instances, and anything below `xlarge` can't sustain 100 MB/s of writes.

The benchmark produces 100,000 events/sec (~1 KB each, ~97 MB/s) and consumes them back, sustained for one hour. 360 million events total. Acks=1 — klite acknowledges each batch after writing to disk.

## Reproduce

[`scripts/bench-aws.py`](https://github.com/klaudworks/klite/blob/main/scripts/bench-aws.py) is a self-contained [uv](https://docs.astral.sh/uv/) Python script. Run it directly — `uv` resolves dependencies inline.

**Requirements:**

- [uv](https://docs.astral.sh/uv/)
- [Terraform](https://www.terraform.io/)
- [Docker](https://www.docker.com/) with buildx
- AWS CLI with a configured profile (`aws configure --profile klite-bench`)
- An SSH key pair (defaults to `~/.ssh/id_ed25519.pub`)

```sh
./scripts/bench-aws.py up    # provision EC2, build & push Docker images
./scripts/bench-aws.py run   # run benchmark (100K rec/s, 1KB, acks=1, 1h)
./scripts/bench-aws.py down  # tear down
```

Flags override defaults:

```sh
./scripts/bench-aws.py run --throughput 200000 --acks -1 --partitions 12
```

Run `./scripts/bench-aws.py --help` for the full list of options.
